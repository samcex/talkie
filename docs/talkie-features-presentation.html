Talkie01

Identity and access

Secure accounts and controlled access

Clerk authenticationUsers sign in and register through Clerk-backed auth screens.
Display namesThe app shows first name, username, full name, or email instead of raw user IDs.
Admin controlsAdmin-only route support exists for managing users and disabled accounts.
Disabled-user blockingToken and user search APIs reject disabled accounts before they can join or be called.
Private identitiesDirect calls use Clerk user IDs internally while showing readable names in the UI.
Vercel-ready configThe app builds from the web project root and deploys from GitHub main.
Security02

Push-to-talk core

Live audio channels powered by LiveKit

Public channelsUsers can join named live rooms such as general.

PIN-protected channelsPrivate rooms are derived from channel name plus PIN.

Push-to-talk micMicrophone access is requested lazily when the user presses the talk button.

Spacebar supportDesktop users can hold Space to transmit.

Connection statusThe channel header shows live, connecting, reconnecting, and offline states.

Recent channelsPreviously joined channels are remembered locally for quick re-entry.

In-channel tools

Everything needed while a channel is live

People panelSee local and remote participants in the room.

Speaking indicatorsActive speakers get visual status and animated VU bars.

Per-user volumeRemote participants can be adjusted independently.

Mute controlsRemote participant audio can be muted locally.

Text chatLive data-channel chat works inside the same room.

Unread badgesChat and replay tabs show unread counts while users are on another tab.

Channel tools05

Replay and feedback

Missed audio can be replayed during the session

Browser-local replayReceived audio clips are recorded in the browser while the channel page is open.
Replay tabUsers can listen back to recent clips without leaving the call.
Auto cleanupClips are capped and revoked locally; they disappear when leaving the channel.
Incoming beepOptional tone plays when someone starts speaking.
VibrationSupported phones can vibrate on incoming speech.
Browser notificationsWhen enabled, backgrounded tabs can show speaking notifications.
Replay06

PWA experience

Installable mobile-first web app

ManifestTalkie ships a web app manifest with app name, icons, and display mode.

Service workerThe app registers a service worker and includes an offline fallback page.

iOS install hintiPhone users get guidance to add the app to the home screen.

Home-screen iconsStandard, Apple touch, and maskable icons are generated.

Mobile shellThe UI is designed around a phone-sized viewport with touch-first controls.

White/red themeHome, channel, and settings screens share the same brand styling.

PWA07

Settings

User-controlled audio and alerts

Output volumeGlobal output level controls remote audio playback.
Beep toggleUsers can turn incoming speech tones on or off.
Vibration toggleVibration can be disabled for devices that support it.
Notification toggleThe app requests browser notification permission only when the user enables it.
Permission-aware UIThe settings screen explains unsupported, blocked, and granted notification states.
Local persistenceSettings are saved in browser local storage.

Current app stack

Next.js React Clerk LiveKit PWA Vercel Local Storage Service Worker

FrontendNext.js app router with client screens for home, channel, settings, auth, and admin.

Realtime audioLiveKit tokens are issued server-side with room grants for audio and data.

IdentityClerk protects the app and supplies user records for names, search, and admin checks.

Local stateRecent channels, contacts, PINs, settings, and replay clips are browser-local.
